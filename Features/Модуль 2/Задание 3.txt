Для классов эквивалентности требуется проверка на значения внутри класса, исключая граничные значения. 
Для техники граничных значений требуется проверка только значений на границе.

Тест-кейс будет содержать 5 значений проверки:
999 - число входящее в класс эквивалентности (до 1000$), не являющееся граничным значением. Проверяем значение скидки равное 0%. 
1000 - граничное значение. Проверяем значение скидки равное 5%.
1999 - число входящее в класс эквивалентности (до 2000$), не являющееся граничным значением. Проверяем значение скидки равное 5%.
2000 - граничное значение. Проверяем значение скидки равное 10%.
2500 - число входящее в класс эквивалентности (от 2000$), не являющееся граничным значением. Проверяем значение скидки равное 10%.
 
999 и 1999 при любом знаке сравнения (< или <=) будут попадать в свой класс эквивалентности.
